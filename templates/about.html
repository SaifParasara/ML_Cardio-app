Machine Learning Architecture

An overview of the analytical pipelines built for cardiovascular risk assessment. While three distinct architectures were evaluated, building predictions across 14 input vectors requires a sophisticated model robust to complex collinearity.

Activated Core Engine

Gradient Boosting Ensemble

A state-of-the-art technique utilizing sequential decision trees. By rigorously fitting each consecutive tree to minimize the residual errors of the prior iteration, the final algorithmic output achieves superb classification boundaries.

Primary Advantages

  • Exceptionally robust on multidimensional tabular datasets
  • Naturally resilient directly against non-linear interactions
  • Low variance compared to singular decision trees

Standard Decision Tree

A fundamental model that iteratively splits data parameters along axes of maximum variance reduction, constructing a highly legible flowchart architecture of logic nodes.

Characteristics

  • High interpretability and visual transparency
  • Extremely rapid inference speed
  • Susceptible to terminal leaf overfitting

Logistic Regression

A reliable baseline statistical blueprint calculating probabilities utilizing the sigmoid function over cross-multiplied dataset feature matrices.

Characteristics

  • Excellent established baseline algorithm
  • Direct coefficient impact transparency
  • Limited scaling against complex data relationships

Training Dataset & Backend Pipeline

Our predictive intelligence is built on an extensive clinical dataset composing of over 70,000 diverse patient records. Each entry tracks 14 distinct physiological attributes—providing a deep metric foundation for drawing correlations in cardiovascular health.

Real-time Inference Architecture

1. Secure Data Ingestion

Raw numerical constraints (such as age, height, and blood pressure) are transmitted from the user interface directly to the Python Flask backend logic, undergoing strict type-safety checks.

2. Dynamic Standardization

The data is passed through a pre-fitted StandardScaler constructed synchronously during original model training. This ensures extreme metric variances are mathematically normalized to prevent algorithmic bias.

3. Algorithmic Inference

The fully constructed 14-dimensional feature vector is processed by the loaded Gradient Boosting Core (RF_model.pkl). The ensemble resolves probability bounds in milliseconds and returns the diagnostic classification.
